# Charter

## 项目目标

长期研究 Claude Code 的 harness 思想，提炼适合 `openclaw` 的运行时增强机制，并最终形成一个我们自己的 `openclaw` 增强插件。

## 成功标准

1. 能明确说明 Claude Code 真正强在哪里
2. 能把这些能力映射到 `openclaw` 当前架构
3. 能输出一套低维护、可渐进实现的增强方案
4. 最终至少做出一个可运行的增强插件原型

## 边界

不做：
- 全量复刻 Claude Code
- 引入与 `openclaw` 现状不匹配的超大规模架构
- 为了“功能看起来多”而增加长期维护负担

优先做：
- harness 核心循环
- 会话状态
- 权限模型
- bridge/gateway 会话控制
- 技能与插件扩展层

## 判断标准

一个能力只有在同时满足下面至少 3 条时，才值得进入 `openclaw`：

1. 明显提升稳定性
2. 明显提升可观测性
3. 明显提升可扩展性
4. 不引入高维护成本
5. 与 `openclaw` 当前结构兼容
