# External Thread Phase 7 Plan

## Purpose

Phase 7 is a narrow operator-and-delivery wave after accepting `33-36`.

It does not reopen broad repo study.
It is designed to support two immediate needs:

- the main thread’s `Milestone 2` clean-host delivery and durable-memory follow-up work
- the longer-range custom-Claw program’s need for exact host/operator contracts instead of vague extension assumptions

## Packet Set

1. `37` OC-16 ContextEngine Slot Lifecycle / Cleanup Symmetry Matrix
2. `38` OC-17 Config Path / ResolvePath / StateDir Contract Matrix
3. `39` CC-14 Memory Hygiene / Pruning / Explainability Matrix
4. `40` SYN-08 SNC Milestone 2 Delivery / Docs Envelope

## Why These Packets

### `37`

`OC-15` closed install/enable/restart reality, but left one sharp clean-host risk open:

- how `contextEngine` slot ownership behaves when plugins are swapped, disabled, updated, or uninstalled

That is the next exact host packet needed for safe SNC delivery and later specialization swaps.

### `38`

`SYN-07` proved that `stateDir` is the real operator boundary.
The next missing precision is:

- where paths are resolved from
- what relative path behavior really is
- how README/demo guidance should talk about `stateDir` and other SNC paths on a clean host

### `39`

The next main-thread durable-memory cut should be diagnostics/control-oriented, not breadth-oriented.

To guide that well, we want one more CC donor packet on:

- stale-memory avoidance
- pruning/dedupe
- operator-legible memory quality signals

### `40`

Once `37` and `38` close, the next synthesis value is not architecture.
It is a release/docs boundary:

- what to tell operators first
- what install path to recommend
- what not to promise yet
- what demo flow is honest and strong
