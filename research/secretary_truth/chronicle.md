# Chronicle

## 当前项目主线

- 这是一个多线程长期项目，目标不是“小修小补 OpenClaw”，而是吃透 `OpenClaw + Claude Code` 的结构、运行机制和可改造边界，然后把结果收敛成可持续、可工程化、可热插拔的 `SNC` 增强层。
- 当前秘书线程负责汇总真相、看管账本、替你调度视角，不抢其他线程的 canonical 文档 ownership。

## 你定下来的硬边界

- `Windows-first`、低维护、按需启动，是本地工具栈的总原则。
- `Qwen` 是默认的大块分析模型。
- `SNC` 默认走热插拔路线；只有当 OpenClaw 宿主真的落后或挡路，才动宿主内部。
- `SNC` 不能和 OpenClaw 现有通用助手能力打架，也不能把所有用户信息都误绑进写作语境。
- 这个线程现在被你当成“秘书线程”来用；我要负责归档、协调、追踪，而不是只会扮演角色。

## 你定义的写作工作流

- `T1` 定题材、卖点、节奏目标和总纲边界
- `T2` 产出可执行的当章细纲包
- `T3` 只按包写正文
- 评审打回后，`T2` 修包，`T3` 重写

这套工作流后面会继续反哺 `SNC` 的多 worker / controller 设计。

## 当前阶段判断

- `Milestone 1` 已经不是嘴上过关，而是有真实 release candidate、验证脚本和打包产物。
- `Milestone 2` 已经从“架构讨论”跨进“产品化补强”：重点不是证明子系统能存在，而是把 worker launch、诊断、clean-host 交付这些硬茬子补齐。
- 研究层已经够厚了。现在真正值钱的是：有节制地继续拆、继续证据化、继续落代码，而不是再堆一座概念山。

## 这层账本的工作方式

- 这里只记关键事实、边界和阶段判断
- 不把 agent 自说自话的大段推理灌进来
- 不反向干预其他线程的工作流
- 需要追根溯源时，再回到 `research/*.md` 和工作树

## 新增秘书职责

- 你在一天结束时可以让我做“今日归档”。
- 归档不追求吞掉所有细节；重点是把秘书该记的东西记住：
  - 你今天给各线程的任务分配
  - 各线程的大致角色和你对它们的说话方式
  - 关键决策、转向、冲突、混乱指令和明显失误
  - 哪些分工有效，哪些安排开始变得别扭
- 这些记录以后不只是复盘材料，也会成为工作流进化的证据库。
