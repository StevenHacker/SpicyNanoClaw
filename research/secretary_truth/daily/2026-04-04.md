## 今天真落地了什么

### 1. `Milestone 2` 已经不是只会投影 worker lane

今天的推进把 worker 这条线从“有结构”继续推向“有结果”：

- 重复 helper launch 不再无脑重排队
- replay hold 进入了 operator 可见层
- 内部 completion event 不再污染 session-state
- 真正的 `sessions_spawn` 结果开始回流到 SNC 自己的 worker state
- launch failure 不再只是一坨工具输出噪音，而是进入了 controller surface

对应文档：

- `research/139_snc_controller_launch_replay_governance_v1.md`
- `research/146_snc_worker_launch_result_host_wiring_v1.md`

### 2. clean-host delivery 终于像个正常产品流程了

今天这条很值钱，因为它把“工程师工作树里的插件”往“普通 OpenClaw 用户也能走”的方向拉了一大截：

- 从干净 host snapshot 起步
- 安装打包后的 `openclaw-snc-0.1.0.tgz`
- 写推荐 base config
- 校验 `contextEngine = snc`
- 走 restart-based 的真实 operator story

对应文档：

- `research/125_snc_clean_host_delivery_rehearsal_v1.md`

### 3. durable memory 不再只靠信仰

今天 durable memory 这条线补上了 operator 侧最缺的东西：

- bounded controls
- prune / stale pressure 的可见性
- projection starvation / saturation 的可见性

也就是说，它开始从“有点用”进化成“能解释为什么这样工作”。

对应文档：

- `research/132_snc_durable_memory_diagnostics_controls_v1.md`

## 今天暴露出来的几条“张力”

这些不是灾难，但以后很可能会成为工作流进化材料。

### 1. 你想要“有血有肉的 agent”，但你同时又非常在意 token 纪律

这不是矛盾到不能共存，但它要求：

- agent 不能只会演人设
- 角色必须落到责任、写入范围、验收标准上
- 否则人格越重，越容易变成贵而散的上下文烟雾

### 2. 你想要跨线程长期记忆，但又不希望 chronicle 反向污染工作流

这条边界今天已经比较清楚了：

- 起居注先记账
- 不主动回灌
- 共享真相单独成层

这个决定是对的，不然现在很容易把一切搞成“记忆系统反客为主”。

### 3. 你在推动多线程协作，但也在防止宿主被过早重写

这条边界今天依然是全局最重要的安全绳：

- 优先插件/slot/hook/context-engine
- 不把 SNC 做成大而全控制平台
- 不抢宿主 scheduler / memory-slot / orchestration ownership

这也是目前整个项目没跑歪的关键原因。
