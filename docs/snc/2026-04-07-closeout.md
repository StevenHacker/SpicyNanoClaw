# SNC Closeout 2026-04-07

This note records the practical state of `SNC` after the first production-facing research and runtime iteration.

The purpose is not archival completeness. The purpose is to give the next developer a concrete starting point.

## Why SNC Exists

The target is not model fine-tuning.

The target is to make the same host plus model combination write stronger serialized fiction by improving:

- continuity
- scene grounding
- recall of story obligations
- resistance to briefing drift and AI-smell

The production novel review showed four recurring failure modes:

1. prose flattening and sentence-pattern homogeneity
2. over-compliance with outline structure at the cost of lived scene pressure
3. weak long-run recall across many chapters
4. no hard gate for obvious prose regressions

That means the main problem is runtime control, not just model capability.

## What Research Actually Changed

The external work that mattered was useful because of mechanism, not because it should be copied literally.

The recurring lesson across `LongWriter`, `Agents' Room`, `StoryWriter`, `Learning to Reason for Long-Form Story Generation`, `WritingBench`, and `Arena-Write` was:

- long-form gains come from decomposition and external control
- writing quality is more reliable under planning plus critique than under naive continuation
- pairwise comparison is better than one static prose score
- continuity and prose-quality should be measured separately

For `SNC`, the practical conclusion was:

`better writing runtime = structured context + structured state + narrow quality pressure + later staged orchestration`

Not:

`just write a bigger prompt`

## What Changed In The Runtime

Current `SNC` on this branch now does five useful things.

### 1. Stable writing guidance is injected only for matching writing sessions

`extensions/snc/index.ts` now adds persistent writing guidance through `before_prompt_build`, but only when:

- the prompt looks like writing work
- the session matches configured `sessionPatterns`

This avoids global leakage into unrelated sessions.

### 2. Session state is no longer just loose continuity notes

`extensions/snc/src/session-state.ts` now persists a more useful shape:

- `scenePacket`
- `styleProfile`
- `chapterState`
- `storyLedger`
- `humanityDiagnostics`

This is still heuristic, but it is already more writing-specific than the older continuity snapshot.

### 3. Context assembly now reflects writing priorities

`extensions/snc/src/engine.ts` builds system context around:

- configured brief and ledger files
- packet files
- structured session snapshot
- quality-watch guidance when recent prose drifted

This is a real step up from plain file dumping.

### 4. State extraction is cleaner

Two practical fixes matter:

- transport metadata such as Telegram `Conversation info` and `Sender` wrappers are stripped before entering state
- style and constraint extraction now favors user instruction over assistant self-talk, so the model's own planning prose is less likely to poison later style state

### 5. Anti-briefing pressure exists now

`extensions/snc/src/humanity-lint.ts` is not a literary critic, but it is a useful lint rail.

It tracks:

- briefing or procedural phrasing
- abstract-label density
- sensory presence
- visible action
- protagonist agency
- canned closing templates
- repeated sentence starts

This currently feeds `Quality watch` pressure into assembly rather than enforcing a rewrite loop.

## What Testing Confirmed

The evaluation result today was directionally strong, but not enough to claim the prose problem is solved.

Reported local validation before production sync:

- workspace `tsc` passed
- focused `vitest` for `SNC` passed
- Docker live smoke ran successfully after the harness repair

The most useful smoke numbers were:

### `modelstudio/qwen3.5-plus`

- `legacy`: overall `13`
- `snc`: overall `20`
- delta: `+7`

Observed gains:

- `statePersisted` moved from `0` to `1`
- context QA improved
- continuation humanity stayed good

Observed limit:

- opening humanity was not uniformly stronger than baseline

### `modelstudio/glm-5`

- `legacy`: overall `13`
- `snc`: overall `22`
- delta: `+9`

Observed gains:

- `statePersisted` moved from `0` to `1`
- context QA improved across all tracked items
- continuation retention improved
- structured state quality registered cleanly

This supports a narrower claim:

`SNC is already better at grounding, persistence, and continuity than stock runtime on the same model.`

It does **not** yet support the stronger claim:

`SNC now reliably produces human-feeling fiction prose.`

## What Production Observation Confirmed

Live production checks against the active fiction project also mattered.

After `SNC` was correctly scoped and configured for the writing agents:

- `gemwriter` and `nukelly` both started persisting structured `SNC` state
- continuity and packet recall improved immediately
- obvious forbidden-term and English leakage was lower in the latest samples
- late-scene collapse into pure chopped-line output was reduced

But the remaining weakness was still visible:

- prose could still feel like an outline being translated
- explanatory phrasing still appeared inside otherwise improved scenes
- continuity improved faster than liveliness

That matches the smoke results.

## Current Judgment

`SNC` is now a credible `v1.5` writing context-engine.

It is good enough to:

- improve continuity under the same model
- preserve more story state across turns
- apply light anti-briefing pressure
- support further runtime experimentation safely

It is not yet a complete serialized-fiction runtime because it still lacks:

- hard output gating
- candidate selection
- narrative compaction
- role-separated planning and critique
- scene-level rewrite control

## Highest-Leverage Next Work

The next step should stay narrow. Do not expand surface area randomly.

### Priority 1: add anti-briefing gate, not just anti-briefing warning

Current `humanity-lint` is advisory. It should become selective enforcement.

Minimum useful upgrade:

- if risk is high, block immediate acceptance
- either route through rewrite or return stronger corrective pressure

### Priority 2: add hard lint for obvious serialized-fiction failures

The production novel review makes this mandatory.

The first hard gates should target:

- forbidden terminology leakage
- English residue
- repeated sentence openings
- repeated template closures
- line fragmentation beyond threshold
- procedural or memo-like phrasing spikes

### Priority 3: move agent work from chapter blob mode to scene mode

The current human-run stack should evolve toward:

- `rosyT1`: boundaries, obligations, no near-prose
- `gemT2`: scene cards instead of pseudo-prose outline
- `nukellyT3`: scene-level drafting, not one-shot full-chapter dump
- `Alice`: split reader-feel review from hard QA review

This is the clearest path to reducing brief-to-prose translation smell.

### Priority 4: own compaction for narrative obligations

Right now `SNC` still delegates compaction.

That is acceptable for continuity scaffolding, but not for long-run fiction control.

Future compaction should preserve:

- unresolved promises
- relationship changes
- emotional outcomes
- still-live hooks
- style prohibitions that must survive many turns

### Priority 5: add candidate comparison before final acceptance

Long-form fiction quality is often easier to detect pairwise than absolutely.

The first practical version can be simple:

- produce two short scene candidates when risk is elevated
- let a critic or rule-weighted selector choose one

## Recommended Short-Term Build Order

If work resumes immediately, use this order:

1. hard lint and selective output gate
2. stronger anti-briefing repair path
3. agent-contract shift toward scene packets
4. narrative compaction
5. candidate selection and critic loop

## Bottom Line

The system no longer needs broad architectural exploration.

The broad answer is already clear:

- keep `OpenClaw` as host
- keep `SNC` as the writing-specific context engine
- improve prose quality through gating, orchestration, and scene-level control

The next wins will come from making `SNC` stricter and more selective, not from making its prompt bigger.
